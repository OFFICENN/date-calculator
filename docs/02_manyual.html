1. アプリの基本操作

タブ機能

2. 年齢計算

使い方

  1. 「生年月日」欄に誕生日を入力
  2. 「計算」ボタンをクリック

表示される情報

項目内容

3. 日数計算

使い方

  1. 「開始日」欄に開始日を入力
  2. 「終了日」欄に終了日を入力（デフォルトは今日）
  3. 「日数を計算」ボタンをクリック

表示される情報

項目内容
日数2つの日付間の合計日数
年月日○年○ヶ月○日 形式
週数合計週数
時間合計時間
分合計分

4. 日付加算

使い方

  1. 「基準日」を入力（デフォルトは今日）
  2. 「加算」または「減算」を選択
  3. 数値と単位（日・週・ヶ月・年）を入力
  4. 「計算」ボタンをクリック

よく使う計算

画面下部に以下のショートカットがあります：

ショートカット内容
1週間後基準日 + 7日
2週間後基準日 + 14日
30日後基準日 + 30日
90日後基準日 + 90日
半年後基準日 + 180日
1年後基準日 + 365日

6. ダークモード

画面右上の丸いボタンで切り替えできます。

  • 🌙（月アイコン）→ ダークモードに切替
  • ☀️（太陽アイコン）→ ライトモードに切替

初回アクセス時は、お使いのPC/スマホのシステム設定に自動追従します。
手動で切り替えた場合、次回以降もその設定が記憶されます。

7. 名前のカスタマイズ

アプリのタイトルに「○○さんの年齢・日数計算」と表示する機能です。3つの方法があります。

方法A：環境変数で固定する（配布・販売用）★推奨

Vercelの環境変数 VITE_OWNER_NAME に名前を設定します。この場合、利用者は名前を変更できません。

→ 詳細は「8. 顧客への配布・販売方法」を参照

方法B：URLパラメータで設定する

https://あなたのURL.vercel.app/?name=田中太郎

URLに ?name=○○ をつけてアクセスすると名前が表示されます。ブラウザに記憶されるので、次回以降はパラメータなしでも表示されます。

※ 環境変数が設定されていない場合のみ有効

方法C：アプリ画面から設定する

画面最下部の「名前を設定する」リンクから設定モーダルを開けます。

※ 環境変数が設定されていない場合のみ表示

8. 顧客への配布・販売方法

仕組み

GitHubリポジトリ（1つだけ） │ │ 同じリポジトリから複数プロジェクトを作成 │ ├── Vercelプロジェクト① あなた用（テスト・開発） │ 環境変数なし → 名前は自由に変更可能 │ URL: date-calculator-dev.vercel.app │ ├── Vercelプロジェクト② 田中さん用 │ VITE_OWNER_NAME = 田中太郎 │ URL: date-calculator-tanaka.vercel.app │ └── Vercelプロジェクト③ 鈴木さん用 VITE_OWNER_NAME = 鈴木花子 URL: date-calculator-suzuki.vercel.app

新しい顧客を追加する手順（約3分）

ステップ1：Vercelで新規プロジェクト作成

  1. https://vercel.com/new を開く
  2. GitHubリポジトリ一覧から date-calculator を選択して「Import」
  3. プロジェクト名 を変更する（例: date-calculator-tanaka）

ステップ2：環境変数を設定

  1. 画面の「Environment Variables」セクションを開く
  2. 以下を入力：
    • Key: VITE_OWNER_NAME
    • Value: 顧客の名前（例: 田中太郎）
  3. 「Add」をクリック

ステップ3：デプロイ

  1. 「Deploy」をクリック
  2. 1〜2分で公開完了
  3. 発行されたURL（例: https://date-calculator-tanaka.vercel.app）を顧客に共有

顧客に表示される画面

  • タイトル：「田中太郎さんの年齢・日数計算」（固定）
  • 名前変更ボタン：非表示（変更不可）
  • フッター：「© OFFICENN」
  • 全機能（年齢計算・日数計算・日付加算・記念日・ダークモード等）が利用可能

既存顧客の名前を変更する場合

  1. Vercelダッシュボードで該当プロジェクトを開く
  2. 「Settings」→「Environment Variables」
  3. VITE_OWNER_NAME の値を変更して「Save」
  4. 「Deployments」タブ → 最新デプロイの「...」→「Redeploy」

注意事項

  • Vercel無料プランのプロジェクト数は無制限
  • GitHubリポジトリは1つだけでOK
  • コードを更新してPushすると、全顧客分が自動で最新版に更新される
  • 顧客ごとに独立したURLが発行される

9. アプリの更新方法

日常の更新フロー

① Cursorでコードを修正 ② ターミナルで以下を実行： git add -A git commit -m "変更内容の説明" git push ③ Vercelが自動検知 → 全プロジェクトが自動再デプロイ ④ 1〜2分で更新完了

特定の顧客だけ更新したくない場合

Vercelダッシュボードで該当プロジェクトの自動デプロイをオフにできます：
Settings → Git → Auto Deploy → 無効化
